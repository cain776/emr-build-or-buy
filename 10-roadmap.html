PART 2 · CHAPTER 10

10. 단계별 로드맵

옵션 C(차트 모듈)를 출발점으로, 3단계에 걸쳐 안과 특화 EMR 카테고리에 진입하는 경로 — 각 단계마다 명시적 Go/No-Go 게이트를 둡니다.

10.1 Stage 1 (2026): 풀스택 보류, 연동 기반 준비

10.2 Stage 2 (2027~2028): 차트 모듈 · 안과 데이터 모델 확장

10.3 Stage 3 (2029~): 안과 특화 표준 EMR 진입 여부 재검토

STAGE 1
현행 유지 + 연동 준비
2026년
자체 EMR 추진 보류. 진행 중인 SOFTCRM 현대화·EyeChartPro·LOOCUS·OMS·VXV 우선 완료. 외부 EMR 연동 인터페이스만 확보(HL7 FHIR R4 기준). 신규 시스템부터 FHIR 호환 데이터 모델 적용 — '우리 데이터를 우리 표준으로'.
STAGE 2
차트 모듈 확장 (옵션 C)
2027~2028년
EyeChartPro CRM에 '차트 영역(SOAP·필차트·이미지·안과 전용 측정 필드)'을 모듈로 추가. 청구는 외부 EMR과 연동. ISMS-P 인증 추진. B&VIIT를 첫 레퍼런스로 운영 데이터 축적. 안과 특화 EMR 카테고리 진입 1단계.
STAGE 3
EMR 본격 진입 재검토
2029년 이후
청구·DUR·진료정보교류까지 포함한 '안과 특화 표준 EMR' 구축 및 EMR 인증 취득. 이 시점에서 시장·여력·도메인 조건을 재평가하여 GO/NO-GO 결정 — 단계 진입을 보장하지 않음.

10.4 병행 운영 전략과 단계별 Go / No-Go 기준

각 Stage 진입 시점에는 다음 게이트를 통과해야 다음 단계로 갑니다. 게이트 미통과 시 해당 단계에 머물거나 후퇴합니다.

병행 운영(coexistence) — Stage 2 핵심 운영 원칙

Stage 2의 차트 모듈 도입은 기존 외부 EMR과 병행 운영됩니다. 진료 중단을 절대 발생시키지 않기 위한 운영 원칙입니다.

  • 마스터 시스템 분담 — 환자·예약·청구는 외부 EMR이 마스터 / 차트·필차트·안과 측정 필드는 신규 모듈이 마스터
  • 데이터 동기화 방향 — 환자 등록·인적사항은 외부 EMR에서 한 번만, 신규 모듈은 환자ID로 조회. HL7 FHIR API + 이벤트 기반 동기화
  • 이중 입력 방지 — 동일 데이터를 두 시스템에 입력하지 않도록 입력 책임 시스템 명확히 정의
  • 장애 시 폴백 절차 사전 정의 — 신규 모듈 장애 시 외부 EMR 단독으로 진료 가능 상태 유지. 폴백 훈련 분기 1회 이상
  • Cut-over는 6개월 누적 후 결정 — 베타 의료기관 운영 데이터 6개월 누적 후 Stage 2 → 3 진입 여부 판단
로드맵의 핵심은 '일정을 지키는 것'이 아니라 '각 게이트를 통과한 후에만 다음 단계로 가는 것'입니다.
2029년에 Stage 3 진입을 보장하지 않습니다 — 6.4 의사결정 4축이 그때 다시 평가될 뿐입니다.
