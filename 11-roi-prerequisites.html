PART 2 · CHAPTER 11

11. ROI 프레임과 선결 과제

정량 ROI는 Stage 2 진입 시점에 별도 회차로 산정합니다. 이 장은 그 산정에 필요한 가치 프레임과, Stage 1(2026)에 반드시 끝내야 할 선결 과제를 정의합니다.

11.1 가치 4축 — 비용 회피 · 신규 매출 · 데이터 자산화 · 글로벌 확장

현 시점 정량 ROI 산정은 근거가 부족합니다(추가 조사 필요 — 11.2). 따라서 본 장에서는 가치를 4개 축의 정성 평가로 정리하고, 정량 산정은 Stage 2 진입 시점(2027~28)에 별도 회차로 갱신합니다.

가치 축내용실현 시점
① 비용 회피외부 EMR 라이선스·청구 모듈 비용, 데이터 추출·연동 위탁비, 신규 안과 클리닉 오픈 시 EMR 도입 패키지 비용 절감Stage 2 후
② 신규 매출EMR 구독료(SaaS) + 부가서비스(렌즈·소모품 발주 수수료, AI 보조 판독, 진료기록 분석) — 경쟁사 모델과 동일 구조Stage 3 후
③ 데이터 자산화B&VIIT 진료 데이터의 통제권 내재화 → LOOCUS AI 학습·검증 파이프라인 안정화 → 모델 품질 상승. 외부 옵션으로는 채울 수 없는 영역Stage 2부터 누적
④ 글로벌 확장성VXV·향후 동남아 안과 체인 진출 시 '진료기록 + CRM + AI 상담' 일체형 솔루션 수출 가능성 — 단 현지 데이터 규제·로컬라이제이션 사전 검토 필수Stage 3 후

이 중 ③ 데이터 자산화가 가장 핵심입니다. ①②④는 다른 옵션(외부 EMR + 부가 솔루션)으로도 부분 충족 가능하지만, ③은 자체 차트 모듈 또는 자체 EMR이 자리잡지 않으면 닫히지 않습니다(6.3 참조).

11.2 정량 ROI 산정을 위한 추가 조사 항목

Stage 2 진입 시점에 정량 ROI를 산정하려면 다음 항목이 사전에 조사·확정되어야 합니다.

  • 외부 EMR(유비케어·비트컴퓨터·이지스헬스케어) 연동 비용 — 1회성 + 연간 + API 호출 단가
  • 차트 모듈 개발 인력·기간 — Stage 2 1차 PoC 기준 산정
  • ISMS-P 인증 취득 비용 — 컨설팅 + 심사 + 사후관리
  • 청구·수가 전문가 인건비 — 상시 운영 1인 기준 연간 비용
  • AI 보조 기능 운영 비용 — 추론 인프라 + 감사로그 보관 + 모델 갱신
  • 외부 EMR과의 병행 운영 시 의료기관 측 추가 부담 (있다면) — 신규 안과 도입 시 무료 패키지화 가능성 검토

11.3 외부 EMR 연동 방식 및 계약 구조 조사

옵션 C(차트 모듈)와 옵션 B(안과+외부 청구 연동) 모두 외부 EMR과의 연동이 전제됩니다. 따라서 2026년 상반기에 외부 EMR 사업자와의 연동 가능성 조사를 끝내야 Stage 2 진입 결정을 할 수 있습니다.

  • 대상 사업자 — 유비케어·비트컴퓨터·이지스헬스케어 중심, 옵션으로 닥터팔레트·오름차트·엣지앤넥스트 등 클라우드 진영
  • 조사 항목 — API 표준(FHIR R4 지원 여부), 데이터 쓰기 권한 범위, 환자ID·예약·청구 데이터 동기화 방향, SLA, 연동 비용 구조
  • 법적 항목 — 데이터 처리 계약(DPA), 환자 동의 책임 분담, 청구 사고 시 책임 소재
  • 운영 항목 — 외부 EMR 장애 시 폴백 절차, 양방향 동기화 충돌 정책

11.4 안과 데이터 표준 모델 정의

차트 모듈의 안과 도메인 데이터(시력·굴절·안압·OCT·각막 토포그래피 등)는 표준이 부재합니다. HL7 FHIR R4를 베이스로 안과 도메인을 매핑하는 자체 표준을 정의해야 합니다.

  • FHIR R4 기반 — Observation, Procedure, ImagingStudy 리소스에 안과 도메인 매핑
  • 측정 데이터 — 양안(OD/OS) 별도 입력 모델, 단위·정밀도 표준
  • 영상 데이터 — DICOM 메타데이터 + 자체 안과 메타데이터 결합 스키마
  • 수술 기록 — LASIK/LASEK/SMILE/ICL 케이스별 BOM 코드 표준
  • 도수 계산기 결과 — 입력값과 술 후 실제 결과의 차이 추적 가능한 구조
의도: 이 표준은 단순 차트 모듈용이 아니라 'B&VIIT가 향후 외부 안과 사업자와 데이터를 주고받을 때의 우리 측 표준'으로 활용됩니다 — 즉 차트 모듈 이전에 LOOCUS, OMS, EyeChartPro의 신규 설계분에 우선 적용합니다.

11.5 청구·수가 전문가 확보, 보안·가명처리 프로세스 설계

Stage 2 진입 전에 다음 두 조직 자산이 사내에 자리잡아야 합니다.

  • 청구·수가 전문가 — 심사평가원 출신 / 의료기관 보험팀 출신 / 보험청구 컨설턴트 1인 이상. 영입에 최소 1년 이상 필요 — 2026년 안에 후보 풀 확보 시작
  • 가명처리(비식별화) 프로세스 — 직접식별자 제거 + 간접식별자 가공 + 결합 위험 평가. 외부 위탁 시 데이터심의위원회 심의 절차
  • 환자 동의 구조 — 일반 진료 동의서와 별도로 2차 활용·AI 학습 동의서 양식 사전 설계
  • 모델 감사로그 — 모든 AI 추론에 사용자·시점·입력·출력 감사로그 보관 정책
  • ISMS-P 인증 추진 계획 — Stage 2 진입 시점에 인증 보유 상태가 되도록 2026~2027 일정 계획

11.6 베타 의료기관 검증 체계, 글로벌 로컬라이제이션 조사

Stage 2 차트 모듈은 B&VIIT를 첫 레퍼런스로 시작하지만, Stage 3 풀스택 진입 검토를 위해서는 추가 안과 1~2곳 베타 운영이 필요합니다. 또한 글로벌(VXV) 확장은 별도 트랙으로 미리 조사해야 합니다.

  • 베타 의료기관 후보 풀 — B&VIIT 외 협력 안과 1~2곳, 인수·신설 후보 안과. Stage 2 진입 시 베타 합의 필요
  • 베타 운영 데이터 표준 — 6개월 운영 후 진료 흐름·청구 정확성·장애 빈도·의료진 만족도 항목별 평가 체계
  • VXV(베트남) 로컬라이제이션 조사 — 현지 데이터 보관 규제(국외 이전 제한 / 로컬 클라우드 의무), 현지 청구 시스템 연동 가능성, 언어·UI 로컬라이제이션 범위
  • 동남아 확장 시나리오 — 한국 솔루션이 그대로 적용되는 영역 vs 현지 재설계 영역 사전 분리
  • 본격 검토 시점은 Stage 2 진입 후 — 2026년 단계에서는 '조사' 수준만 진행
